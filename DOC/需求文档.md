做一个“科研助手”是一个非常有价值但也极具挑战的项目！🎯 针对计算机/AI方向的研究生群体（结合通常的科研痛点），我为你梳理了一份详细的需求文档。

我们可以将需求分为**核心功能（Must-have）**、**进阶功能（Nice-to-have）以及技术架构需求**三个维度来规划。

-----

### 🚀 一、 核心功能需求 (MVP 阶段)

这部分旨在解决“读不完、找不到、写不出”的核心痛点。

#### 1\. 智能文献检索与追踪 (The Hunter 🕵️‍♂️)

  * **多源聚合搜索：** 支持连接 arXiv, Google Scholar, IEEE Xplore, DBLP 等API。
  * **每日ArXiv推送：** 根据用户设定的关键词（如 `Time Series`, `Multi-Agent`, `LLM`），自动筛选当日最新论文，生成一句话总结（TL;DR）。
  * **引用网络分析：** 给定一篇论文，能自动找出其“前世今生”（重要的参考文献和引用了它的SOTA论文）。

#### 2\. 深度阅读与分析 (The Reader 📚)

  * **PDF 解析与全文对话 (RAG)：** 上传PDF，支持对全文、图表、公式进行提问。
      * *Case:* “这篇文章使用了什么数据集？Metrics 是多少？”
  * **创新点提取：** 自动提取论文的 `Motivation` (动机), `Method` (方法), `Contribution` (贡献) 和 `Limitations` (不足)。
  * **跨文档对比：** 允许同时选中3-5篇论文，让助手生成横向对比表格（如：模型架构对比、实验结果对比）。

#### 3\. 写作辅助与润色 (The Writer ✍️)

  * **学术润色 (Polishing)：** 提供“降重”、“学术化表达”、“逻辑增强”三种模式。
      * *Input:* "The model is good at finding patterns."
      * *Output:* "The proposed model demonstrates superior efficacy in identifying underlying patterns."
  * **LaTeX 格式化：** 能够将纯文本、图片或手写公式直接转换为标准的 LaTeX 代码。
  * **参考文献生成：** 自动生成 BibTeX 格式，且能检查引用格式是否符合会议标准（如 CVPR/ICLR 模板）。

-----

### 🌟 二、 进阶功能需求 (差异化竞争点)

这部分功能让你的助手更像一个“Agent”而不仅仅是工具。

#### 1\. 实验辅助 (The Coder 💻)

  * **Code-Paper 互查：** 输入 GitHub 链接和论文 PDF，分析代码是否复现了论文中的所有模块。
  * **环境配置向导：** 针对 `requirements.txt` 报错提供解决方案，或自动生成 Dockerfile。
  * **伪代码转 Python：** 截图论文中的 Algorithm 伪代码，直接生成可运行的 Python 代码框架。

#### 2\. 知识库管理 (The Librarian 🗄️)

  * **Zotero/Mendeley 双向同步：** 能够读取用户的 Zotero 库，并在对话中直接引用库中的条目。
  * **个人科研知识图谱：** 根据用户阅读过的论文，自动构建可视化的知识图谱，展示概念之间的联系。

#### 3\. 投稿辅助 (The Reviewer ⚖️)

  * **AI 预审稿：** 模拟 Reviewer 的视角，对用户的论文草稿提出尖锐的批评意见（Rebuttal模拟训练）。
  * **期刊/会议推荐：** 根据论文主题和质量，推荐合适的投稿去向及截止日期提醒。

-----

### ⚙️ 三、 技术架构需求 (非功能性需求)

考虑到你可能使用 Java/Spring Boot 或 Python 进行开发，建议采用**多智能体 (Multi-Agent)** 架构。

| 模块 | 技术选型建议 | 备注 |
| :--- | :--- | :--- |
| **LLM 基座** | GPT-4o / Claude 3.5 (云端) <br> DeepSeek-Coder / Llama 3 (本地) | 科研对逻辑推理要求高，建议支持模型切换。 |
| **Agent 编排** | **LangGraph** (Python) 或 **LangChain4j** (Java) | 用于管理复杂的“搜索-\>阅读-\>总结”工作流。 |
| **向量数据库** | Milvus / Chroma / Pinecone | 存储论文的 Embedding，用于 RAG。 |
| **解析工具** | Nougat (Meta) / MinerU | 专门用于解析学术论文中的复杂数学公式和双栏排版。 |
| **前端交互** | Streamlit / Next.js + React Markdown | 需要良好的 LaTeX 渲染支持 ($...$)。 |

### 💡 一个具体的 Agent 工作流示例

如果你使用 **LangGraph** 来设计，可以设计如下的图：

1.  **用户输入：** “帮我调研一下 Time Series Foundation Models 的最新进展。”
2.  **Planner Agent：** 分解任务 -\> 1. 搜索ArXiv近3个月论文; 2. 筛选高引论文; 3. 总结方法论; 4. 生成报告。
3.  **Search Agent：** 调用 ArXiv API 获取 Top 10 列表。
4.  **Reader Agent：** 并行读取 10 篇 PDF 的 Abstract 和 Conclusion。
5.  **Writer Agent：** 汇总信息，生成 Markdown 综述报告，并附上 BibTeX。
6.  **Human Feedback：** 用户查看报告，提出修改意见。
